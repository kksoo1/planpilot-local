
# PlanPilot Local MVP 수동 회귀 테스트 결과

이 문서는 `docs/manual-test-checklist.md` 기준으로 사람이 직접 수행한 MVP 수동 회귀 테스트 결과를 기록하기 위한 문서다.

현재 문서는 결과 기록용 템플릿에서 시작했으며, 일부 수동 테스트 결과가 기록되어 있다.

Codex 정리 메모:

- 최근 기록 기준으로 Today, 전체 업무, 프로젝트, 설정, IndexedDB, 금지 기능 핵심 항목은 통과로 기록되었다.
- 이전 보류 항목이었던 추천 업무 rule-based 기준과 완료 업무 표시/숨김 항목도 통과로 갱신되었다.
- 테스트 개요, 발견 이슈, 후속 조치를 현재 상태에 맞게 정리했다.
- 실패/보류 항목이 없으므로 MVP 완료로 판정한다.
- Codex는 사용자가 통과로 기록하지 않은 항목을 임의로 통과 처리하지 않는다.

- 보류: 환경, 데이터, 정책 미정 등으로 판정을 미룬다.

## JSON export 수동 확인 결과

이 섹션은 MVP 완료 이후 추가된 JSON 데이터 내보내기 기능의 별도 수동 확인 결과다.

- 테스트 일자: 2026-05-29
- 테스트 브랜치: `auto/dev-loop`
- 테스트 대상 커밋: `3d209e2 Add JSON data export`
- 전체 결과: 성공 경로 통과, 실패 경로 보류

통과로 기록된 항목:

- [통과] 설정 화면에 데이터 내보내기 버튼이 표시된다.
- [통과] 버튼 클릭 시 JSON 파일이 다운로드된다.
- [통과] 파일명은 `planpilot-backup-YYYY-MM-DD.json` 형식이다.
- [통과] JSON 최상위에 `format`, `schemaVersion`, `exportedAt`, `tasks`, `projects`, `appSettings`가 포함된다.
- [통과] 내보내기 후 기존 앱 데이터가 유지된다.
- [통과] 성공 메시지가 표시된다.

보류/확인 필요 항목:

- [보류] 실패 상황은 수동 재현하지 못했다.
- [확인 필요] 실패 상황에서 기존 IndexedDB 데이터가 유지되는지는 실패 경로 재현 절차가 마련된 뒤 확인한다.

후속 조치:

- JSON export 실패 경로 재현 방법 또는 개발용 확인 절차를 별도 작업으로 정리한다.
- JSON import/복원 기능은 이번 확인 범위에 포함하지 않는다.
