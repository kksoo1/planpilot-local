# PlanPilot Local 수동 테스트 체크리스트

이 문서는 자동 테스트가 아니라, 리팩터링 또는 기능 변경 후 사람이 실제 화면에서 하나씩 확인하는 수동 테스트 체크리스트다.

테스트 전 확인:

- [ ] 테스트 전 `npm run build`가 성공했다.
- [ ] 테스트 대상 변경에 `src/App.css`, DB schema, `package-lock.json` 변경이 포함되지 않았는지 확인했다.
- [ ] 테스트 중 서버 API, 로그인, 클라우드 동기화, 알림 권한 요청, Capacitor 기능을 새로 사용하지 않는다.

## 1. 앱 시작/초기 데이터

- [ ] 앱을 처음 열었을 때 하단 탭에 `오늘`, `업무`, `프로젝트`, `설정`이 보인다.
- [ ] IndexedDB 초기 상태에서 기본 프로젝트가 생성된다.
- [ ] 새로고침 후 기본 프로젝트가 유지된다.
- [ ] 화면 이동 시 앱이 흰 화면이나 오류 화면으로 멈추지 않는다.

## 2. 오늘 화면

- [ ] `오늘` 탭으로 이동할 수 있다.
- [ ] 전체 업무 수가 현재 저장된 전체 업무 개수와 일치한다.
- [ ] 완료 업무 수가 `done` 상태 업무 개수와 일치한다.
- [ ] 오늘 이전 마감일을 가진 미완료 업무가 지난 마감 업무에 표시된다.
- [ ] 완료된 지난 마감 업무는 지난 마감 업무에 표시되지 않는다.
- [ ] 오늘부터 7일 이내 마감일을 가진 미완료 업무가 7일 이내 마감 업무에 표시된다.
- [ ] 완료된 예정 업무는 7일 이내 마감 업무에 표시되지 않는다.
- [ ] 추천 업무가 우선순위와 마감일 기준으로 표시된다.
- [ ] 추천 업무가 없을 때 빈 상태 메시지가 어색하지 않게 표시된다.
- [ ] 오늘 화면의 업무 카드에서 프로젝트 이름이 올바르게 표시된다.

## 3. 전체 업무 화면

- [ ] `업무` 탭으로 이동할 수 있다.
- [ ] `새 업무 추가` 버튼으로 업무 추가 form을 열고 닫을 수 있다.
- [ ] 제목만 입력해 업무를 추가할 수 있다.
- [ ] 제목, 메모, 마감일, 중요도, 프로젝트를 입력해 업무를 추가할 수 있다.
- [ ] 빈 제목으로 업무가 추가되지 않는다.
- [ ] 추가한 업무가 전체 업무 목록에 표시된다.
- [ ] 업무 수정 버튼으로 수정 form이 표시된다.
- [ ] 업무 제목, 메모, 마감일, 중요도, 프로젝트를 수정할 수 있다.
- [ ] 수정 취소 버튼이 수정 전 상태로 돌아간다.
- [ ] 업무 삭제 버튼 클릭 시 확인 창이 표시된다.
- [ ] 삭제를 확인하면 업무가 목록에서 사라진다.
- [ ] 삭제된 업무가 오늘 화면과 프로젝트 통계에도 남지 않는다.
- [ ] 완료 버튼으로 업무를 완료 처리할 수 있다.
- [ ] 완료된 업무를 다시 미완료로 되돌릴 수 있다.
- [ ] 업무 검색은 제목 기준으로 동작한다.
- [ ] 검색어를 지우면 전체 조건에 맞는 업무가 다시 표시된다.
- [ ] `완료 업무 표시`를 끄면 완료 업무가 숨겨진다.
- [ ] `완료 업무 표시`를 켜면 완료 업무가 다시 표시된다.
- [ ] 프로젝트 필터에서 특정 프로젝트를 선택하면 해당 프로젝트 업무만 보인다.
- [ ] 프로젝트 필터를 `전체 프로젝트`로 되돌리면 전체 업무가 보인다.
- [ ] `마감일 빠른 순` 정렬에서 마감일이 있는 업무가 날짜순으로 표시된다.
- [ ] 마감일이 없는 업무는 마감일 빠른 순 정렬에서 뒤쪽에 표시된다.

## 4. 프로젝트 화면

- [ ] `프로젝트` 탭으로 이동할 수 있다.
- [ ] 프로젝트 이름만 입력해 프로젝트를 추가할 수 있다.
- [ ] 프로젝트 이름과 설명을 입력해 프로젝트를 추가할 수 있다.
- [ ] 빈 이름으로 프로젝트가 추가되지 않는다.
- [ ] 프로젝트 수정 버튼으로 이름과 설명을 수정할 수 있다.
- [ ] 프로젝트 수정 취소 버튼이 수정 전 상태로 돌아간다.
- [ ] 기본 프로젝트는 삭제되지 않는다.
- [ ] 기본 프로젝트 카드에는 삭제 불가 상태가 이해 가능하게 표시된다.
- [ ] 업무가 연결된 프로젝트는 삭제되지 않는다.
- [ ] 업무가 있는 프로젝트 카드에는 삭제 불가 상태가 이해 가능하게 표시된다.
- [ ] 업무가 없는 사용자 프로젝트는 확인 후 삭제된다.
- [ ] 프로젝트별 전체 업무 수가 실제 연결 업무 수와 일치한다.
- [ ] 프로젝트별 완료 업무 수가 실제 완료 업무 수와 일치한다.
- [ ] 프로젝트별 미완료 업무 수가 실제 미완료 업무 수와 일치한다.

## 5. 설정 화면

- [ ] `설정` 탭으로 이동할 수 있다.
- [ ] 테마 값이 표시된다.
- [ ] 언어 값이 표시된다.
- [ ] AI Provider 값이 표시된다.
- [ ] 알림 사용 여부가 표시된다.
- [ ] 첫 실행 완료 상태가 표시된다.
- [ ] 설정 화면에서 알림 권한 요청 팝업이 뜨지 않는다.
- [ ] 설정 화면에서 서버 또는 계정 로그인을 요구하지 않는다.

## 6. 데이터 유지

- [ ] 업무를 추가한 뒤 새로고침해도 업무가 유지된다.
- [ ] 프로젝트를 추가한 뒤 새로고침해도 프로젝트가 유지된다.
- [ ] 업무 수정 내용이 새로고침 후에도 유지된다.
- [ ] 프로젝트 수정 내용이 새로고침 후에도 유지된다.
- [ ] 완료/미완료 상태가 새로고침 후에도 유지된다.
- [ ] 브라우저를 종료하고 다시 열어도 IndexedDB에 저장된 업무와 프로젝트가 유지된다.
- [ ] 설정 값이 새로고침 후에도 유지된다.

## 7. 리팩터링 회귀 테스트

- [ ] `TodayView` 분리 후 오늘 화면의 통계, 추천, 지난 마감, 예정 업무 표시가 유지된다.
- [ ] `TasksView` 분리 후 업무 추가, 수정, 삭제, 완료 토글, 검색, 필터, 정렬이 유지된다.
- [ ] `ProjectsView` 분리 후 프로젝트 추가, 수정, 삭제 방지, 통계 표시가 유지된다.
- [ ] `SettingsView` 분리 후 설정 값 표시가 유지된다.
- [ ] `TaskCard` 분리 후 업무 카드의 라벨, 버튼, 프로젝트 이름 표시가 유지된다.
- [ ] `ProjectCard` 분리 후 프로젝트 카드의 설명 fallback, 통계, 버튼 상태가 유지된다.
- [ ] `TaskForm` 분리 후 업무 추가 form과 수정 form의 입력/저장/취소 동작이 유지된다.
- [ ] `ProjectForm` 분리 후 프로젝트 추가 form과 수정 form의 입력/저장/취소 동작이 유지된다.
- [ ] `taskFilters` 분리 후 검색, 완료 업무 표시, 프로젝트 필터, 마감일 정렬 결과가 이전과 같다.
- [ ] `taskDates` 분리 후 지난 마감 업무와 7일 이내 마감 업무 계산이 이전과 같다.

## 8. 날짜 처리 정책 테스트

- [ ] 마감일이 비어 있는 미완료 업무는 지난 마감 업무에 표시되지 않는다.
- [ ] 마감일이 비어 있는 미완료 업무는 7일 이내 마감 업무에 표시되지 않는다.
- [ ] invalid `dueDate`가 있는 업무는 날짜 기반 목록에서 제외된다.
- [ ] 오늘 이전 마감일의 미완료 업무는 지난 마감 업무에 표시된다.
- [ ] 오늘 마감 업무는 지난 마감 업무에 표시되지 않는다.
- [ ] 오늘 마감 업무는 7일 이내 마감 업무에 표시된다.
- [ ] 오늘부터 7일 이내 마감일의 미완료 업무는 7일 이내 마감 업무에 표시된다.
- [ ] 7일 범위를 벗어난 미래 마감 업무는 7일 이내 마감 업무에 표시되지 않는다.
- [ ] 완료된 지난 마감 업무는 지난 마감 업무에 표시되지 않는다.
- [ ] 완료된 예정 업무는 7일 이내 마감 업무에 표시되지 않는다.
- [ ] 추천 업무는 단순 날짜 목록과 별개로 우선순위와 마감일 기준 정렬을 유지한다.
- [ ] 날짜 유틸 통합 후에도 추천 업무의 기존 표시 개수와 정렬 의도가 유지된다.

## 9. 금지 기능 확인

- [ ] 네트워크 탭 또는 코드 검토 기준으로 서버 API 호출이 없다.
- [ ] `localStorage`를 사용하지 않는다.
- [ ] 로그인 또는 계정 생성 UI가 없다.
- [ ] 클라우드 동기화 UI 또는 동작이 없다.
- [ ] 브라우저 알림 권한 요청이 발생하지 않는다.
- [ ] Android notification 권한 요청이 없다.
- [ ] Capacitor 관련 UI, 설정, 패키징 동작이 없다.
- [ ] 개인정보가 외부 서버로 전송되는 동작이 없다.
